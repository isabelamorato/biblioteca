# Skills de Auditoria por Dominio

Referencia de quais skills usar na Fase 5 (Auditoria) baseado na stack do projeto.

## Mapeamento

| Dominio | Skills |
|---------|--------|
| Supabase | `supabase-audit-rls`, `supabase-audit-rpc`, `supabase-best-practices`, `supabase-postgres-best-practices` |
| React/Next.js | `react-best-practices`, `vercel-react-best-practices`, `nextjs-app-router-fundamentals`, `nextjs-seo` |
| Tailwind | `tailwind-4-docs`, `tailwind-v4-shadcn`, `tailwind-design-system` |
| Chrome Extension | `chrome-extension-wxt`, `chrome-extension-development` |
| Monorepo | `monorepo-management` |
| Animacoes | `framer-motion`, `accessible-motion` |
| Acessibilidade | `accessibility` |
| Cloudflare | `cloudflare`, `wrangler` |
| Svelte | `svelte5-best-practices` |
| E2E Testing | `e2e-testing-patterns` |
| TanStack Query | `accelint-tanstack-query-best-practices` |

## Como atualizar

Quando instalar novas skills tecnicas, adicione-as na tabela acima no dominio correspondente.
Para listar skills instaladas: `ls ~/.claude/skills/` e `ls ~/.agents/skills/`
